# Платформа учёта коммунальных платежей (ЖКУ)

## Описание проекта

Веб-платформа для автоматизации учёта жилищно-коммунальных услуг с интеграцией Telegram-бота. Система позволяет управляющей компании вести учёт жителей, счётчиков, показаний, начислений и платежей, а жителям — самостоятельно подавать показания и контролировать баланс через Telegram.

## Используемые технологии

### Backend

| Технология | Версия | Назначение |
|---|---|---|
| **Python** | 3.11+ | Основной язык разработки |
| **FastAPI** | — | Асинхронный веб-фреймворк (REST API + SSR) |
| **SQLAlchemy 2.0** | async | ORM для работы с базой данных |
| **PostgreSQL** | 16 | Реляционная СУБД |
| **Alembic** | — | Миграции базы данных |
| **Uvicorn** | — | ASGI-сервер |
| **Pydantic** | v2 | Валидация данных и конфигурация |
| **JWT (PyJWT)** | — | Аутентификация через cookie-токены |
| **bcrypt (passlib)** | — | Хеширование паролей |
| **APScheduler** | — | Планировщик задач (напоминания) |
| **ReportLab** | — | Генерация PDF-квитанций |
| **openpyxl** | — | Импорт/экспорт Excel-файлов |

**Архитектурные решения:**
- Полностью асинхронная обработка запросов (async/await)
- Репозиторный паттерн для доступа к данным (`app/repositories/`)
- Сервисный слой для бизнес-логики (`app/services/`)
- JWT-аутентификация через HTTP-only cookies (TTL 30 минут)

### Frontend

| Технология | Назначение |
|---|---|
| **Jinja2** | Серверный рендеринг HTML-шаблонов |
| **Bootstrap 5.3** | CSS-фреймворк для адаптивной вёрстки |
| **Custom CSS (Stitch)** | Авторский дизайн в editorial-стиле |
| **Manrope / Inter** | Шрифты (заголовки / основной текст) |

**Дизайн-система Stitch Editorial:**

- Pill-бейджи с мягкими фонами
- Таблицы с uppercase-заголовками 11px без жёстких рамок
- Компонентный подход: пагинация, карточки дашборда, формы

### Telegram-бот

| Технология | Назначение |

## Модель данных

```
User (оператор/админ)
  │
  ├── Resident (житель) ──── Address (адрес)
  │       │                      │
  │       │                      ├── Meter (счётчик) ── MeterReading (показание)
  │       │                      │
  │       │                      └── Charge (начисление) ── Tariff
  │       │
  │       └── Payment (платёж) ── PaymentPeriod
  │
  └── UtilityService (услуга: вода, газ, электричество...)
         │
         └── Tariff (тариф: цена за единицу, период действия)
```

**Основные сущности:**
- **Resident** — житель (ФИО, телефон, лицевой счёт, Telegram ID)
- **Address** — адрес (город, улица, дом, квартира, площадь)
- **Meter** — счётчик (серийный номер, тип услуги, статус)
- **MeterReading** — показание (значение, потребление, источник, статус проверки)
- **Charge** — начисление (потребление × тариф за период)
- **Payment** — платёж (сумма, способ оплаты, квитанция)
- **Tariff** — тариф (цена за единицу, срок действия)
- **PaymentPeriod** — расчётный период (год, месяц, статус)

## Руководство пользователя: Оператор / Администратор

### Вход в систему

1. Откройте браузер и перейдите по адресу `http://localhost:8000`
2. Введите логин и пароль на странице входа
3. После успешного входа вы попадёте на **Дашборд**

### Дашборд

На главной странице отображается сводная информация:
- Количество зарегистрированных жителей
- Показания счётчиков за текущий месяц
- Общая задолженность
- Платежи за текущий месяц
- Список непроверенных показаний (последние 10)
- Текущие тарифы по услугам

### Управление жителями

**Просмотр списка:**
- Перейдите в раздел **«Жители»** в боковом меню
- Используйте строку поиска для фильтрации по ФИО
- Список разбит на страницы (20 записей)

**Добавление жителя:**
1. Нажмите кнопку **«Добавить жителя»**
2. Заполните форму: ФИО, телефон, лицевой счёт
3. Укажите адрес: город, улица, дом, квартира, площадь
4. Нажмите **«Сохранить»**

**Импорт из Excel:**
1. Перейдите в раздел **«Импорт»**
2. Скачайте шаблон для импорта жителей
3. Заполните шаблон данными
4. Загрузите файл и нажмите **«Импортировать»**
5. Система покажет результат: сколько создано, пропущено, ошибок

**Просмотр карточки жителя:**
- Нажмите на имя жителя в списке
- На карточке: основные данные, адреса, статистика начислений/платежей, задолженность, последние платежи

### Управление счётчиками

**Просмотр:**
- Раздел **«Счётчики»** — список всех счётчиков с адресами и статусами

**Добавление счётчика:**
1. Нажмите **«Добавить счётчик»**
2. Выберите адрес и тип услуги
3. Укажите серийный номер
4. Нажмите **«Сохранить»**

**Деактивация:**
- Нажмите кнопку деактивации рядом со счётчиком

### Работа с показаниями

**Просмотр показаний:**
- Раздел **«Показания»**
- Фильтры: год, месяц, статус проверки, поиск по жилцу

**Проверка показаний:**
- Непроверенные показания выделены в списке
- Нажмите **«Одобрить»** для подтверждения корректности
- Нажмите **«Отклонить»** для удаления некорректного показания

**Импорт показаний из Excel:**
1. Перейдите в **«Импорт»** → вкладка **«Показания»**
2. Скачайте шаблон
3. Заполните: серийный номер счётчика, значение, год, месяц
4. Загрузите файл

### Начисления

**Просмотр:**
- Раздел **«Начисления»**
- Фильтры по году и месяцу
- Отображаются: жилец, адрес, услуга, потребление, сумма

**Расчёт начислений:**
1. Выберите период (год, месяц)
2. Нажмите **«Рассчитать»**
3. Система автоматически:
   - Берёт проверенные показания за период
   - Применяет текущие тарифы
   - Создаёт записи начислений
4. Вы увидите статистику: сколько начислений создано, сколько пропущено

**Закрытие периода:**
- Нажмите **«Закрыть период»**
- После закрытия изменение начислений невозможно

**Формирование квитанции:**
- На странице начислений нажмите **«Квитанция»** напротив жилца
- Скачается PDF с детализацией: услуги, потребление, тарифы, итого

### Платежи

**Просмотр:**
- Раздел **«Платежи»** — список всех платежей с поиском

**Регистрация платежа:**
1. Нажмите **«Добавить платёж»**
2. Выберите жилца (отобразится его задолженность)
3. Укажите: сумму, дату, способ оплаты (наличные / карта / перевод / онлайн)
4. Опционально: номер квитанции и примечание
5. Нажмите **«Сохранить»**

### Тарифы

**Просмотр:**
- Раздел **«Тарифы»** — тарифы сгруппированы по услугам
- Отображаются: действующие и архивные тарифы

**Создание тарифа:**
1. Нажмите **«Добавить тариф»**
2. Выберите услугу
3. Укажите цену за единицу и дату вступления в силу
4. Предыдущий активный тариф автоматически закроется

### Отчёты

**Отчёт по периоду:**
1. Раздел **«Отчёты»** → выберите год и месяц
2. Сводка: начисления по услугам, общие итоги
3. Кнопка **«Экспорт в Excel»** для скачивания

**Список должников:**
1. Раздел **«Отчёты»** → **«Должники»**
2. Список жилцов с задолженностью > 0
3. Кнопка **«Экспорт в Excel»** для скачивания

### Типичный рабочий процесс оператора (ежемесячный)

1. **1–20 числа:** Жители подают показания через бот или оператор вносит вручную/импортирует из Excel
2. **20–25 числа:** Оператор проверяет показания в разделе «Показания» (одобряет/отклоняет)
3. **25–28 числа:** Оператор рассчитывает начисления за период и закрывает период
4. **По мере поступления:** Оператор регистрирует платежи
5. **При необходимости:** Формирует отчёты и квитанции

## Руководство пользователя: Житель (Telegram-бот)

### Начало работы

1. Найдите бота в Telegram (ссылку предоставит управляющая компания)
2. Нажмите **«Старт»** или отправьте команду `/start`
3. Бот запросит ваш **лицевой счёт** — введите его
4. Подтвердите ваши данные (ФИО, адрес)
5. После регистрации вам станет доступно главное меню

### Главное меню

После регистрации в боте доступны следующие кнопки:

| Кнопка | Команда | Описание |
|---|---|---|
| 👤 Профиль | `/profile` | Ваши данные, адрес, количество счётчиков |
| 💰 Баланс | `/balance` | Начисления по услугам и задолженность |
| 📋 История оплат | `/payments` | Последние 10 платежей |
| 📊 Подать показания | `/readings` | Подача показаний счётчиков |
| 📈 Тарифы | `/tariffs` | Текущие тарифы по услугам |
| ❓ Помощь | `/help` | Справка по работе с ботом |

### Подача показаний счётчиков

1. Нажмите **«📊 Подать показания»** или отправьте `/readings`
2. Бот покажет список ваших счётчиков — выберите нужный
3. Введите текущее показание (целое число или с десятичной точкой)
4. Бот проверит:
   - Показание не может быть меньше предыдущего
   - При аномально высоком потреблении бот предупредит вас
5. Подтвердите показание
6. Готово — показание отправлено на проверку оператору

**Важно:** Подавайте показания **до 20-го числа** каждого месяца. Бот пришлёт напоминание автоматически.

### Просмотр баланса

1. Нажмите **«💰 Баланс»** или отправьте `/balance`
2. Бот покажет:
   - Начисления по каждой услуге (вода, газ, электричество и т.д.)
   - Общую сумму начислений
   - Общую сумму оплат
   - Текущую задолженность или переплату

### Просмотр истории платежей

1. Нажмите **«📋 История оплат»** или отправьте `/payments`
2. Бот покажет последние 10 платежей:
   - Дата платежа
   - Сумма
   - Способ оплаты

### Просмотр профиля

1. Нажмите **«👤 Профиль»** или отправьте `/profile`
2. Бот покажет:
   - ФИО и лицевой счёт
   - Телефон
   - Адрес (с площадью квартиры)
   - Количество счётчиков
   - Текущую задолженность

### Просмотр тарифов

1. Нажмите **«📈 Тарифы»** или отправьте `/tariffs`
2. Бот покажет по каждой услуге:
   - Название услуги
   - Цену за единицу (руб./м³, руб./кВт·ч и т.д.)
   - Дату вступления тарифа в силу

### Автоматические уведомления

Бот автоматически присылает напоминания:
- **20-го числа** каждого месяца — напоминание о подаче показаний
- **5-го числа** каждого месяца — уведомление о задолженности (если есть долг)

### Отмена действия

Если вы начали какую-то операцию и хотите отменить — нажмите **«Отмена»** или отправьте `/cancel`.

## Запуск проекта

### Требования
- Docker и Docker Compose
- (Опционально) ngrok для внешнего доступа

### Запуск

```bash
# Перейти в директорию проекта
cd communal-payments

# Запустить контейнеры
docker compose up -d --build

# Проверить статус
docker compose ps

# Посмотреть логи
docker logs communal-payments-app-1 --tail 50
```

Приложение будет доступно по адресу: `http://localhost:8000`

### Внешний доступ (ngrok)

```bash
ngrok http 8000
```

### Перезапуск после изменений

```bash
docker compose restart app
```
